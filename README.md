# คำแนะนำในการรันโปรเจค FriendFind

หากทำการรันแอปพลิเคชันผ่าน Expo Go บนโทรศัพท์มือถือแล้วพบปัญหา **"Network time out"** หรือ **"The Internet connection appears to be offline"** 

สาเหตุไม่ได้เกิดจากโค้ดของแอปพลิเคชัน แต่เกิดจากระบบ Network หรือ Firewall ของเครื่องคอมพิวเตอร์ทำการบล็อกการเชื่อมต่อ (ทำให้มือถือไม่สามารถดึงไฟล์แอปจากคอมพิวเตอร์ไปแสดงผลได้)

## สิ่งที่ต้องทำก่อนรันแอปพลิเคชัน (สำคัญมาก)
เครื่องคอมพิวเตอร์ทุกเครื่องที่จะรันแอปนี้ **ต้องติดตั้งและเปิดฐานข้อมูลก่อน** มิฉะนั้นแอปจะ Login ไม่ได้:
1. เปิดโปรแกรม **XAMPP** แล้วกด Start ที่ Apache และ MySQL
2. เข้าไปที่ `http://localhost/phpmyadmin`
3. สร้าง Database ใหม่ชื่อว่า **`study_match_db`**
4. ทำการ Import ไฟล์ SQL (เช่น `schema.sql` และ `dummy_data.sql` ที่อยู่ในโฟลเดอร์โปรเจค) เพื่อสร้างตารางข้อมูล
5. เปิด Terminal เข้าไปที่โฟลเดอร์ `backend/backend` แล้วรัน `npm install` (ถ้ายังไม่เคยทำ) จากนั้นรัน **`npm start`** เพื่อเปิดเซิฟเวอร์ API

## วิธีการรัน Frontend ที่แนะนำ (รับประกันว่ารันได้ 100%)

### วิธีที่ 1: รันผ่าน Web Browser (ไม่ต้องใช้มือถือ)
เป็นวิธีที่ง่ายที่สุดและไม่ต้องกังวลเรื่อง Network หรือ Firewall
1. รัน Backend ทิ้งไว้ก่อน (ตามขั้นตอนด้านบน)
2. เปิด Terminal ใหม่เข้าไปที่ `frontend`
3. รันคำสั่ง `npx expo start`
4. ในหน้าต่าง Terminal ของ Frontend ให้กดปุ่ม **`w`** บนคีย์บอร์ด
5. แอปพลิเคชันจะเปิดขึ้นมาบน Web Browser ทันที

### วิธีที่ 2: รันผ่าน Android Emulator
หากมี Android Studio ติดตั้งอยู่ในเครื่อง สามารถรันผ่าน Emulator ได้โดยตรง
1. เปิด Android Emulator ให้เรียบร้อย
2. รัน Backend: `cd backend` -> `npm start`
3. รัน Frontend: `cd frontend` -> `npx expo start`
4. ในหน้าต่าง Terminal ของ Frontend ให้กดปุ่ม **`a`** บนคีย์บอร์ด
5. แอปจะถูกติดตั้งและเปิดบน Emulator ทันที

### วิธีที่ 3: หากต้องการแสกนผ่านมือถือจริง (แนะนำวิธี Tunnel)
หากอาจารย์ต้องการแสกน QR Code เพื่อทดสอบบนโทรศัพท์มือถือจริง และพบปัญหา Network Time out สามารถแก้ได้ง่ายๆ 2 วิธีดังนี้:

**วิธีที่ 3.1: ใช้โหมด Tunnel (แนะนำ - ไม่ติด Firewall แน่นอน)**
เป็นการจำลองเส้นทางเชื่อมต่อผ่านอินเทอร์เน็ตโดยตรง:
1. รันคำสั่ง `npx expo start --tunnel` ในโฟลเดอร์ frontend
2. ระบบจะสร้าง QR Code ใหม่ ให้ใช้แอป Expo Go แสกนได้เลย (วิธีนี้ไม่ต้องใช้ Wi-Fi วงเดียวกันก็สามารถรันได้)

**วิธีที่ 3.2: รันผ่าน LAN (วิธีปกติ)**
หากรันด้วย `npx expo start` แบบปกติ กรุณาตรวจสอบให้แน่ใจว่า:
1. **Wi-Fi เดียวกัน:** โทรศัพท์มือถือและคอมพิวเตอร์ **ต้องเชื่อมต่อ Wi-Fi ชื่อเดียวกัน**
2. **Network Profile:** ใน Windows ต้องตั้งค่า Wi-Fi เป็น **Private Network** (หากเป็น Public Network ระบบจะบล็อกการแสกน)
3. **Firewall:** ไม่ได้ถูกบล็อกโดย Windows Defender Firewall
